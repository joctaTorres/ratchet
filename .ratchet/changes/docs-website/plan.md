# docs-website

## Why

Ratchet has no documentation website; its reference material lives only in the
README. The project's `documentation` standard mandates Diátaxis Reference docs
at the repository-root `docs/` directory, but `docs/` does not exist yet. This
change stands up a Docusaurus site that renders `docs/` as a browsable website
with a distinctive landing page, deployed automatically on Cloudflare Pages when
`main` updates — so future content has a home and users can read the docs in a
browser. The content/doc-generation pipeline is explicitly out of scope.

## What Changes

- Add a standalone Docusaurus (classic preset, TypeScript) app at `website/`
  with its own `package.json` and lockfile, **not** added to the root pnpm
  workspace. Implements `features/docs-website/site-structure.feature` and
  `features/docs-website/isolation.feature`.
- Configure the docs plugin to source content from `../docs`, serve reference
  docs under `/docs`, autogenerate the sidebar, and set `onBrokenLinks: 'throw'`.
  Implements `site-structure.feature` and `build-and-deploy.feature`.
- Add a custom landing page at `/` (`website/src/pages/index.tsx`) with a
  machined dark-first theme, monospaced display headline, hero with the ratchet
  logo and CTAs, and a three-card capability summary. Implements
  `features/docs-website/landing-page.feature`.
- Enable i18n with `defaultLocale: 'en'` and `locales: ['en']`, served at the
  root with no locale prefix and no locale switcher until a second locale is
  added. Implements `features/docs-website/i18n.feature`.
- Seed `docs/intro.md` so the build has content and `/docs/intro` resolves.
  Implements `features/docs-website/docs-seed.feature`.
- Add `website/build/` and `website/node_modules/` to root `.gitignore`.
  Implements `isolation.feature`.
- Document the website (local dev, build, Cloudflare Pages git-integration
  settings) in `website/README.md`, and link the docs site from the repository
  `README.md`. Implements `build-and-deploy.feature` and `docs-seed.feature`.
- Cloudflare Pages deploy uses **git integration** (dashboard-configured): the
  repository builds on push to `main`. No changes to `ci.yml` and no GitHub
  Actions deploy job.

## Design

**Isolation is the central decision.** `website/` is a standalone npm package
with its own lockfile, deliberately excluded from `pnpm-workspace.yaml`
(`packages: ['.']` stays unchanged). This keeps Docusaurus's large dependency
tree out of the CLI's `pnpm install --frozen-lockfile`, lint, test, coverage,
e2e, and the gated provenance npm publish — none of those CI steps change. The
trade-off (a second lockfile / install) is accepted in exchange for keeping the
publish pipeline untouched and the failure domains separate. Cloudflare Pages'
per-project **root directory** setting (`website`) makes a subfolder app a clean
fit for git-integration builds.

**Content sourcing.** The docs plugin `path` points to `../docs` with an
autogenerated sidebar (`{ type: 'autogenerated', dirName: '.' }`), so files added
to `docs/` later by the out-of-scope pipeline appear with zero sidebar
maintenance. `routeBasePath: 'docs'` serves reference docs at `/docs` and frees
`/` for the custom landing page.

**Landing page & theme.** A single self-contained `src/pages/index.tsx` using
Docusaurus's `Layout`, themed via Infima CSS variables in `src/css/custom.css`:
dark-first "machined" palette with one sharp accent, hairline rules, monospaced
display headline paired with a humanist sans body (not Inter/Roboto), and one
orchestrated staggered page-load reveal. Card copy is adapted from the README's
"Why ratchet?" and batch sections to stay accurate. The ratchet logo asset is
copied into `website/static/`.

**i18n.** Configured but minimal (YAGNI): `defaultLocale: 'en'`, `locales:
['en']`. The default locale builds at the root with no `/en/` prefix; the locale
dropdown is omitted while a single locale exists. The translation workflow
(`docusaurus write-translations`, adding a locale) is documented in
`website/README.md` so a language can be added later without rework.

**Deploy.** Cloudflare Pages git integration is authorized once in the dashboard;
settings are recorded in `website/README.md`: production branch `main`, root
directory `website`, build command `pnpm install && pnpm build`, output
`website/build`, build variable `NODE_VERSION=22` (matching CI). Preview
deployments for PRs come for free. `onBrokenLinks: 'throw'` makes a broken
internal link fail the Cloudflare build rather than ship a broken site.

**Out of scope:** the content/doc-generation pipeline, any `ci.yml` changes,
search, and versioning.

**Standards.** This change follows the `documentation` standard: it creates the
mandated repository-root `docs/` directory, documents the new website surface as
Reference material in `website/README.md`, and updates `README.md` to link the
docs site (Task group 6 below, mandatory and blocking). The `multi-agent-support`
standard does not apply: this change adds no agent-facing skill, command, or
generated artifact and special-cases no agent — the site and its content are
agent-neutral.
