# Documentation website

The ratchet documentation website. A [Docusaurus](https://docusaurus.io) app
that renders the repository-root `docs/` directory as a browsable site with a
custom landing page, deployed on Cloudflare Pages.

This package is **standalone**: it is its own pnpm workspace root
(`website/pnpm-workspace.yaml`) with its own lockfile and is **not** part of the
repository-root pnpm workspace. The CLI package's install, CI gates, and npm
publish never see the site's dependencies.

## Content source

Reference content is **not** stored in this directory. The docs plugin sources
markdown from the repository-root `docs/` directory (`path: '../docs'` in
`docusaurus.config.ts`) and serves it under the `/docs` route. The sidebar is
autogenerated from the files under `docs/`, so new pages appear automatically
with no sidebar edits. The site root (`/`) is the custom landing page
(`src/pages/index.tsx`).

## Local development

Run from the `website/` directory:

```bash
pnpm install        # install dependencies (own lockfile)
pnpm start          # start the dev server with hot reload at http://localhost:3000
pnpm build          # build the static site into website/build
pnpm serve          # serve the built site locally
pnpm typecheck      # type-check the TypeScript sources
```

`pnpm build` fails on a broken internal link (`onBrokenLinks: 'throw'`), so a
dead link is caught at build time rather than shipped.

## Internationalization

i18n is enabled with English as the only locale (`defaultLocale: 'en'`,
`locales: ['en']` in `docusaurus.config.ts`). The default locale builds at the
site root with no `/en/` path prefix, and no locale switcher is shown while a
single locale exists.

To add a language without restructuring the site:

1. Generate the translation files for the default locale:
   ```bash
   pnpm write-translations
   ```
2. Add the new locale code to `locales` in `docusaurus.config.ts`
   (e.g. `locales: ['en', 'pt-BR']`).
3. Add a locale dropdown to the navbar by adding a
   `{type: 'localeDropdown'}` item under `themeConfig.navbar.items`.
4. Translate the content under `i18n/<locale>/` and the markdown under that
   locale's docs directory.

## Deployment — Cloudflare Workers Builds (static assets)

The site is deployed via **Cloudflare Workers Builds** with the static-assets
handler (no Worker script): Cloudflare builds and deploys on every push, so the
live site updates when `main` is updated. The build runs on Cloudflare's side —
it is not part of the repository's GitHub Actions CI
(`.github/workflows/ci.yml`). The contract is declared in `website/wrangler.toml`
(the `[assets]` block) and mirrored in the dashboard.

Configure these settings once in the Workers project (Workers & Pages → your
project → Settings → Builds):

| Setting                 | Value                       |
| ----------------------- | --------------------------- |
| Production branch       | `main`                      |
| Root directory          | `website`                   |
| Build command           | `pnpm install && pnpm build`|
| Build output directory  | `build`                     |
| Deploy command          | `npx wrangler deploy`       |
| Build variable          | `NODE_VERSION` = `22`       |

`wrangler` runs inside the root directory (`website`), so `wrangler.toml`'s
`[assets].directory = "./build"` resolves to `website/build`.
`not_found_handling = "404-page"` serves `build/404.html` on a miss (Docusaurus
emits it). Pull requests receive Cloudflare preview deployments automatically.
No secrets are stored in the repository; the git connection is authorized once in
the Cloudflare dashboard.
